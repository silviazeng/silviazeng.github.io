# Design context — Ever Branching (silviazeng.github.io)

A personal Quartz digital garden. Two deliberate visual worlds: a **terminal garden**
(the home + notes) and **paper exhibits** (immersive, art-directed Lab artifacts).

## Design Context

### Users
Readers of Silvia's notes and Lab pieces — split across three strands: AI / recsys
(career), history & art (Getty, Medici, Italian politics), and personal/living
reflection. The art & museum pieces are **bilingual EN / 中文**, so CJK rendering is a
first-class concern, not an afterthought.

### Brand personality
Considered · terminal-literary · handmade. The home reads like a hacker's commonplace
book — a command prompt that happens to be set in a literary serif. Warmth comes from
type and restraint, never from decoration.

### Aesthetic Direction
- **Home & notes** — dark "Terminal Garden": monospace structure + serif reading voice,
  green / amber / lavender accents on near-black. Dark because it's a focused reading UI.
- **Lab artifacts** — warm cream "museum / manuscript" exhibits: high-contrast serif on
  paper, art-directed per piece. Light because they're printed-catalogue analogues.
- Anti-reference: generic AI dashboard (rounded cards + icon-above-heading + cyan-on-dark).

### Design Principles
1. **Two type regimes, never a third.** Every page declares Core *or* Exhibit (below).
2. **Tokens over hardcoding.** Reference the regime's named stack; never inline a one-off
   family name in a component.
3. **Monospace is the structural voice, serif is the reading voice** — don't blur the roles.
4. **Art-direct exhibits within the sanctioned palette.** Character comes from layout,
   color, and paper texture — not from adding a new font.
5. **Bilingual-safe by default.** Never strip the CJK face (see anti-drift rules).

## Type System (the contract — keep pages on it)

Canonical Core tokens live in `quartz.config.ts` (theme.typography) and are exposed as
CSS vars in `quartz/styles/custom.scss` (`--font-mono` / `--font-display` / `--font-prose`).

### Regime A — Core (Terminal Garden)
Used by: the whole Quartz site + standalone **Latin** artifacts (`recsys-roadmap.html`,
`medici-guide.html`).

| Role | Font | Used for |
|---|---|---|
| `--font-mono`    | **JetBrains Mono** (400/500) | UI, nav, meta, code, post & lab titles, tags, panels |
| `--font-display` | **Playfair Display** (400/500, italic) | headings h1–h6, hero, pillar names |
| `--font-prose`   | **Source Serif 4** (400, italic) | body prose, quotes, descriptions, subtitles |

Standalone Core page `<link>`: `JetBrains Mono:wght@400;500 + Playfair Display + Source Serif 4:ital`.

### Regime B — Exhibit (paper / museum)
Used by: bilingual Getty guides (`getty/villa-guide`, `getty/center-guide`,
`getty/floorplans`, `getty/villa-floorplans`) and the Latin-only `italia-politica.html`.
All four Getty pages share an identical `:root` palette block — keep it identical.

| Role | Font | Used for |
|---|---|---|
| Latin display / emphasis | **Cormorant Garamond** (400/500/700, italic) | titles, `em`, map labels |
| Chinese serif (read)      | **Noto Serif SC** (400/600/700) | long-form 中文 body & headings on *guide* pages |
| Chinese sans (navigate)   | **Noto Sans SC** (400/500/700) | 中文 UI, legends, toasts, badges on *map* pages |

- **Guide pages** (long read) lead with Noto Serif SC body + load all three faces.
- **Map pages** (navigation) lead with Noto Sans SC body; they legitimately omit Noto Serif SC.
- `italia-politica.html` loads **only Cormorant Garamond** (no CJK on that page).

## Anti-drift rules (why the type kept getting messy)

The main site is tokenized, but the standalone Lab pages each hardcode fonts + their own
Google Fonts `<link>`, so they drift independently every time a new one is built. To stop it:

1. **A new standalone artifact adopts ONE regime wholesale** — copy the `<link>` and the
   role mapping from an existing page of that regime. Don't improvise a stack.
2. **Never write bare `font-family: sans-serif;` or `serif;`** on an Exhibit page — it
   strips the CJK face, so Chinese falls back to a system font and the page looks
   inconsistent. Use the regime's named stack (`'Noto Sans SC', sans-serif`). *(This was
   the exact bug fixed on 2026-06-11.)*
3. **`font-family: inherit;` is allowed only on form controls** (`button`, `input`) to
   inherit the body font. Never use it as a list fallback — `'X', inherit` is invalid CSS.
4. **Don't add a typeface to give a page "character."** Removed `IM Fell English` from
   `italia-politica.html` — it was loaded but never applied (dead weight). 7 faces → 6,
   two clean regimes.
